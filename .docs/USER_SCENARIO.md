# 사용자 시나리오 (User Scenario) - 마이링크 (MyLink)

이 문서는 마이링크 서비스에서 발생할 수 있는 모든 기능적 흐름과 예외 상황을 **'사용자는 [목적]하기 위해 [행동]한다'**의 형태를 차용하여 체계적으로 정리한 시나리오 문서입니다.

---

## 1. 방문자(Visitor) 플로우

### 1.1 정상 접속 및 콘텐츠 소비 (링크 클릭)
* 사용자는 소유자(크리에이터, 개발자 등)의 다양한 채널 정보를 파악하기 위해 웹 브라우저에서 `mylink.com/닉네임` 주소로 접속한다.
* 사용자는 언제 어디서든 편하게 내용을 확인하기 위해 모바일, 태블릿, 데스크탑 기기 환경에 맞춰 자동으로 조절되는 반응형 화면(UI)을 경험한다.
* 사용자는 평소 관심 있던 소유자의 개인 포트폴리오나 외부 사이트로 진입하기 위해 화면 하단에 직관적으로 목록화되어 있는 링크 아이템(버튼) 중 하나를 클릭한다.
* 사용자는 편안하게 목적지 페이지 내용을 소비하기 위해 클릭과 동시에 새 탭(또는 창)으로 열린 목적지 웹사이트 구조를 만난다. *(※ 이때 백그라운드 시스템은 해당 개별 링크의 '클릭 카운트'를 1회 상승시켜 데이터베이스에 기록한다.)*

### 1.2 예외 상황 탐색
* 사용자는 잘못된 정보가 입력되었거나 이미 탈퇴한 계정의 페이지임을 인지하기 위해, 존재하지 않는 닉네임으로 접속했을 때 노출되는 **"존재하지 않는 프로필입니다"**라는 안내 팝업 및 화면을 본다.
* 사용자는 현재 접근할 수 있는 추가 콘텐츠가 없음을 인지하기 위해, 소유자가 가입 후 등록한 외부 링크가 단 하나도 없을(빈 화면) 때 노출되는 **"아직 등록된 링크가 없습니다"** 텍스트 메시지를 본다.

---

## 2. 소유자(Owner) 플로우

### 2.1 회원가입, 온보딩 및 세션 관리
* 사용자는 복잡한 이메일 가입 절차를 피하고 빠르게 마이링크 서비스를 시작하기 위해 메인 페이지에서 구글(Google) 소셜 로그인 버튼을 클릭한다.
* 사용자는 자신의 고유 방문 주소를 설정하기 위해 구글 인증 직후 제공되는 온보딩 화면에서 본인이 사용할 **'닉네임'**을 수동으로 직접 타이핑한다.
* 사용자는 희망하는 닉네임을 확실히 선점하여 아이덴티티를 굳히기 위해, 닉네임 입력창 하단의 실시간 닉네임 중복 검사 시스템 통과 메시지를 확인한다.
* 사용자는 추후 다른 계정으로 접속하거나 공용 PC에서 보안을 유지하기 위해, 대시보드에 위치한 **로그아웃** 버튼을 클릭하여 구글 연동 세션을 안전하게 종료한다.

### 2.2 메인 프로필 관리 및 공유 (인라인 편집)
* 사용자는 방문자들에게 자신의 현재 상태를 매력적으로 어필하기 위해 대시보드 최상단의 '닉네임' 및 '한 줄 소개(bio)' 영역을 직접 클릭하여 텍스트 **인라인 편집(Inline Edit)** 모드로 전환한다.
* 사용자는 수정한 프로필 텍스트를 서버 데이터베이스에 안전하게 반영하기 위해 텍스트 입력 영역 우측에 활성화된 **'✔️(체크 표시/저장 아이콘)'**을 필수적으로 클릭한다.
* 사용자는 완성된 자신의 마이링크 페이지를 인스타그램 인포나 블로그 등 외부 SNS에 공유하기 위해, 대시보드 상단에 배치된 **'내 링크 주소 복사(Copy)'** 아이콘을 클릭하여 클립보드에 주소를 담는다.

### 2.3 링크 데이터 추가 및 자동 파비콘 연동
* 사용자는 새롭게 홍보하고 싶은 외부 채널 목록을 덧붙이기 위해 관리자 대시보드 중앙의 **'링크 추가'** 버튼을 클릭하여 최상단에 새 빈 카드를 생성한다.
* 사용자는 새로 추가된 링크에 사용자들이 직관적으로 이해할 수 있는 제목(Title)과 실제 연결될 도착 URL을 배정하기 위해 링크 카드의 텍스트 빈 공간(Placeholder)을 눌러 직접 타이핑한다.
* 사용자는 작성한 데이터를 최종 확정 짓기 위해 인라인 영역 옆에 나타난 **'✔️(저장 아이콘)'**을 눌러 수동으로 저장을 마친다.
* 사용자는 추가한 외부 사이트 링크가 방문자에게 더 높은 신뢰감을 주는 시각적 리소스를 제공하는지 검토하기 위해, 링크가 정상 등록됨과 동시에 Гу글 API가 추출한 대상 **URL 맞춤형 파비콘(Favicon)** 이미지가 해당 링크 아이콘 자리에 자연스럽게 렌더링된 것을 확인한다.

### 2.4 기존에 등록된 링크 수정
* 사용자는 이미 올린 외부 채널 주소가 최근에 바뀌었거나 오타를 발견하여 바로잡기 위해 이미 존재하는 링크 리스트 대상 카드의 텍스트 영역을 마우스로 클릭한다.
* 사용자는 바뀐 내용을 덮어씌워 완전히 보존하기 위해 새 내용 입력 후 활성화된 **'✔️(저장 아이콘)'** 버튼을 눌러 인라인 수정을 마친다.
* 사용자는 변경 사항이 올바르게 적용되었는지에 대한 의문을 해소하기 위해, 화면의 전면 새로고침 없이도 즉시 바뀐 내용이 링크 리스트 UI에 실시간으로 표시되는 것을 목격한다.

### 2.5 링크 삭제 안전 절차 (컨펌 보호)
* 사용자는 홍보 기간이 끝났거나 더 이상 필요성이 사라진 특정 링크 뷰를 치우기 위해, 대상 링크 카드 맨 우측 끝부분에 위치한 **'휴지통(삭제 아이콘)'** UI 버튼을 누른다.
* 사용자는 조작 실수로 귀중하게 구성한 링크 데이터가 즉각적으로 허무하게 날아가는 것을 예방하기 위해, 휴지통 클릭 시 화면 전면에 경고 팝업으로 가로막으며 나타나는 **"정말 삭제하시겠습니까?"** 모달창을 대면한다.
* 사용자는 링크 파기를 완전히 수행하기로 마음먹기 위해 해당 모달 창 내 경고 요소를 충분히 숙지한 후 **'확인'** 버튼을 최종 클릭한다.

### 2.6 링크 클릭별 성과 측정 (조회수 확인)
* 사용자는 자신이 제공한 수많은 외부 이동 링크들 중 어느 채널과 이슈가 대중들의 클릭 및 관심을 가장 많이 이끌어냈는지 파악하기 위해, 대시보드 내부의 각 개별 링크 카드 한구석에 작게 표시된 **'클릭 카운트(방문자가 해당 링크를 클릭해 조회한 총 횟수)'** 배지 숫자를 확인한다.
