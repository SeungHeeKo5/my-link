# 프로젝트 기획서 (PRD) - 마이링크 (MyLink)

## 1. 프로젝트 개요
* **프로젝트명:** 마이링크 (MyLink)
* **목적:** 크리에이터, 인플루언서, 마케터 등이 자신의 다양한 SNS 채널, 포트폴리오, 타 사이트 등을 하나의 링크로 모아서 간편하게 공유할 수 있는 '멀티 링크 프로필 서비스'를 구축.
* **대상 사용자 (Target Audience):**
    * 일차적으로는 자신의 깃허브, 블로그, 포트폴리오 등을 모아서 공유하고 싶은 **개발자를 주 사용자로 타겟팅**합니다. (단, 지나치게 복잡하거나 무거운 개발자 특화 기능은 배제하고 심플하게 유지)
    * 단일 프로필 링크만으로 다양한 채널을 홍보하고 싶은 크리에이터 (유튜브, 인스타 등)
    * 개인 이력서나 링크를 모아서 전달해야 하는 프리랜서 및 취준생

## 2. 예상 기술 스택 (Tech Stack)
* **프론트엔드 (UI):** React / Next.js (현재 환경 기반), shadcn/ui 기반 컴포넌트 활용
* **인증 및 백엔드 (BaaS):** Firebase (Authentication: Google Social Login 사용, Firestore database 등)
    * **DB 모델링 정책:** Firestore 사용자 문서를 구성할 때 `displayName` 필드가 필수로 포함되어야 합니다. 또한 하위 링크 관련 데이터들은 문서 내 **서브컬렉션(Sub-collection)** 구조를 적용하여 구성합니다.

## 3. 핵심 기능 목록

### 3.1 필수 기능 (초기 MVP 배포 목표)
1. **Google 소셜 로그인 (Firebase Auth)**
    * 구글 계정을 이용한 간편 회원가입 및 로그인 지원. 별도 이메일/비밀번호 가입 제외.
2. **고유 프로필 링크(URL) 할당**
    * 별도의 영문 핸들 없이 사용자의 **닉네임** 자체를 주소로 사용하여 방문 링크를 설정합니다. (예: `mylink.com/닉네임` 형식).
3. **프로필 기본 정보 관리**
    * 이미지 업로드 기능 없이, 닉네임과 한 줄 자기소개 내용만 작성 및 수정하도록 지원합니다.
4. **링크 관리 시스템 (CRUD)**
    * **타 사이트 이동용 링크** 추가, 이름 변경, 대상 URL 수정, 삭제.
    * **파비콘(Favicon) 자동 연동:** 링크(URL)를 입력하면 **구글 파비콘 API(Google API)**를 호출하여 해당 사이트의 파비콘을 링크 아이콘으로 로드.
    * *(※ 링크 노출 순서 변경(드래그 앤 드롭 등) 및 활성화/비활성화 토글(숨김) 기능은 제외하여 핵심 기능에만 집중)*
5. **퍼블릭 프로필 웹페이지**
    * 외부 방문자가 URL을 클릭했을 때 접속되는 모바일 최적화 디스플레이 뷰.

### 3.2 제외 대상 기능 (Out of Scope)
* **기존의 모든 선택 기능(디자인 테마 변경 등) 제외**
* **방문자 통계 분석 보드 제외** (단, 나중에 추가 기능 형태로 '개별 링크 단위의 클릭 조회수' 카운트 기능만 덧붙일 예정)
* **어드민 내 모바일 화면 실시간 미리보기 기능 제외**

## 4. 기능 상세 설명 및 정책

### 4.1 로그인 및 온보딩 (온보딩 플로우)
* 사용자가 처음 구글 로그인을 완료하면 URL로 사용될 **닉네임을 유저가 직접 수동 입력**하는 단계를 거칩니다.
* 입력한 닉네임은 실시간 중복 검사를 통과해야 하며, 확정된 닉네임이 사용자의 프로필 고유 URL(`mylink.com/닉네임`)이 됩니다.

### 4.2 관리자 대시보드 (어드민 페이지)
* **화면 레이아웃:** 
    * 단일 화면에 링크 관리 리스트와 프로필 설정 폼을 심플하게 스크롤 구조로 배치합니다. (실시간 모바일 뷰 미리보기 영역 등은 제공하지 않음)
* **UI/UX (인라인 편집 및 shadcn/ui 적용):**
    * 닉네임과 한 줄 프로필 소개, 그리고 링크 리스트 등의 수정은 클릭하여 그 자리에서 바로 텍스트를 바꿀 수 있는 **인라인 편집(Inline Edit)** 방식을 사용합니다.
    * 그 외 추가 폼 영역 등은 shadcn/ui의 Input, Button, Card 등을 활용하여 깔끔하게 통일합니다.

### 4.3 퍼블릭 방문자 페이지 (엔드 유저 뷰)
* 사용자가 등록한 요소만이 심플하게 출력되는 반응형 웹페이지입니다.
* 모바일 환경에 최적화된 넓이와 버튼 크기를 가집니다.
* 최상단: 구글 계정 연동된 프로필 이미지 없이 **닉네임, 한 줄 소개글**만 깔끔하게 노출
* 하단: 직관적인 외부 링크 버튼 목록 (클릭 시 새 창 열림 등)

## 5. 데이터베이스 모델링 (Firestore 구조)

### 5.1 컬렉션: `users`
* 사용자의 기본 정보와 고유 닉네임을 저장하는 최상위 컬렉션입니다.
* **문서 ID (Document ID):** Firebase Auth UID
* **필드 (Fields):**
    * `nickname` (string): 프로필 접속 URL로 사용될 고유 식별 주소.
    * `displayName` (string): *(필수 포함)* 화면 상에 노출될 닉네임/이름.
    * `bio` (string): 한 줄 프로필 소개글.

### 5.2 서브 컬렉션: `links`
* 경로 구조: `users/{uid}/links`
* 해당 사용자가 등록한 외부 이동 링크들을 저장하는 1:N 서브 컬렉션입니다.
* **문서 ID (Document ID):** Firestore 자동 생성 ID
* **필드 (Fields):**
    * `title` (string): 링크 버튼에 표시될 표시 타이틀
    * `url` (string): 도착 지점인 타겟 웹사이트 주소
    * `faviconUrl` (string): 구글 API를 통해 확보한 파비콘 이미지 링크
    * `createdAt` (timestamp): 생성 일시 (리스트 노출 순서 정렬 기준)
