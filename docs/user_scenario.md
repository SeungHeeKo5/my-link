# MyLink - 사용자 시나리오 (User Scenario)

## 1. 방문자 (Visitor) 시나리오: 마이링크 페이지 방문 및 상호작용
- 방문자는 **소유자의 통합 프로필과 링크 정보를 확인하기 위해** 소유자가 공유한 마이링크 고유 URL(예: `mylink.com/@username`)에 접속한다.
- 방문자는 **데이터를 기다리는 동안 화면이 멈춘 것처럼 보이지 않게 하기 위해** 짧은 순간 표시되는 스켈레톤(Skeleton) 로딩 UI를 확인한다.
- 방문자는 **소유자가 등록한 최신 링크들을 파악하기 위해** 직관적으로 나열된 프로필과 링크 목록을 위아래로 스크롤하며 탐색한다.
- 방문자는 **자신이 원하는 특정 사이트(포트폴리오, SNS 등)로 이동하기 위해** 목록에 있는 링크 버튼 중 하나를 클릭한다.
- 방문자는 **원래 보고 있던 마이링크 페이지를 잃어버리지 않기 위해** 새 탭(New Tab)으로 열린 웹사이트에서 탐색을 진행한다.

## 2. 소유자 (Owner) 시나리오: 링크 추가/수정/삭제 및 통계 확인 과정

### 2.1. 관리자 페이지 접근 및 프로필 수정 (인라인 에디팅 & 명시적 저장)
- 소유자는 **자신의 마이링크 정보를 관리하기 위해** 사이트에 접속하여 '구글 소셜 로그인' 버튼을 클릭해 인증한다.
- 소유자는 **자신의 닉네임이나 소개글을 수정하기 위해** 모바일 또는 데스크탑 환경에서 프로필 화면에 보이는 텍스트를 터치하거나 클릭하여(인라인 에디팅) 새로운 내용을 타이핑한다.
- 소유자는 **수정한 내용을 취소하고 이전 상태로 되돌리기 위해** 텍스트 상자 주변의 '취소' 버튼을 누르거나 바깥을 클릭(Blur)한다.
- 소유자는 **수정된 텍스트를 데이터베이스에 최종 반영하기 위해** 텍스트 입력창 근처에 나타난 **'저장(Save)' 버튼을 명시적으로 클릭**한다.

### 2.2. 링크 추가
- 소유자는 **방문자에게 새로운 웹사이트를 공유하기 위해** 화면 하단이나 고정된 영역에 있는 '+ 새로운 링크 추가' 버튼을 클릭한다.
- 소유자는 **어떤 사이트인지 명확히 안내하기 위해** 새롭게 생성된 빈 링크 블록을 클릭하여 제목(Title)과 URL 주소를 차례로 입력하고 **'저장' 버튼을 클릭**한다.
- 소유자는 **링크 URL 입력 시 실수를 방지하기 위해** 올바른 형식(`https://...`)이 아니면 경고를 확인하고 수정한다.
- 소유자는 **해당 사이트의 시각적 식별성을 높이기 위해** URL 저장 후 자동으로 불러와지는 파비콘(아이콘) 이미지가 정상적으로 적용되는지 확인한다.

### 2.3. 링크 수정 및 비공개 처리
- 소유자는 **기존에 등록한 링크의 제목이나 URL 주소를 변경하기 위해** 링크 블록의 텍스트 영역을 클릭하여 내용을 고친 후 **'저장' 버튼을 클릭**한다.
- 소유자는 **잠시 동안 방문자에게 특정 링크를 보여주지 않기 위해** 링크 블록 우측에 위치한 '토글(Toggle) 스위치'를 클릭하여 비공개(Off) 상태로 전환한다.

### 2.4. 링크 삭제
- 소유자는 **더 이상 유효하지 않거나 불필요해진 링크를 완전히 제거하기 위해** 해당 링크 블록 영역에 나타나는 '삭제(휴지통) 아이콘'을 클릭한다.
- 소유자는 **실수로 귀중한 링크 데이터를 지우는 것을 방지하기 위해** 화면에 나타나는 "정말 삭제하시겠습니까?" 확인 팝업창에서 '삭제' 버튼을 한 번 더 클릭한다.

### 2.5. 링크 클릭 통계 (Analytics) 확인
- 소유자는 **어떤 링크가 방문자들에게 가장 많이 소비되었는지 파악하기 위해** 대시보드 화면에 나타난 각 링크별 통계 수치를 확인한다.
- 소유자는 **효율적인 프로필 운영을 위해** 오직 "해당 링크가 방문자에 의해 클릭된 횟수(카운트)" 만을 직관적인 숫자 형태로 확인한다.
