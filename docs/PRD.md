# MyLink - 기능 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명:** 마이링크 (MyLink)
- **목적:** 여러 개의 흩어진 링크(소셜 미디어, 포트폴리오, 블로그, 이메일 등)를 하나의 페이지에서 직관적으로 제공하고 관리할 수 있는 개인용 통합 프로필 서비스 구축.
- **대상 사용자:** 오직 개발자 본인(단일 사용자)이 관리하고 운영하는 링크 노출 및 포트폴리오 관리용 어드민 플랫폼.

## 2. 핵심 기능 목록

### 2.1. 필수 기능 (MVP - Minimum Viable Product)
- **회원 인증 시스템:** Firebase Authentication을 활용한 **오직 '구글 소셜 로그인' 전용** (본인 외 접근 통제)
- **프로필 관리:** 프로필 이미지, 닉네임, 간단한 소개글 등록 및 수정 (**명시적 '저장' 버튼이 포함된 인라인 에디팅** 적용)
- **링크 관리 (CRUD):** 
  - 새로운 링크 추가 (URL, 타이틀 입력 후 명시적 저장)
  - 등록된 링크 수정 및 삭제
  - 링크 공개/비공개 토글 기능
- **디자인 시스템 구축:** `shadcn/ui` 라이브러리를 활용한 빠르고 일관된 컴포넌트 기반 UI/UX 구축
- **링크 노출 페이지 (프론트엔드):** 생성된 링크들을 방문자에게 보여주는 단일 랜딩 페이지 구성

### 2.2. 선택 기능 (고도화 및 추가 개선 사항)
- **드래그 앤 드롭 정렬:** 사용자가 링크의 노출 순서를 마우스로 쉽게 변경
- **자동 아이콘(파비콘) 추출:** 링크 주소 입력 시 구글 API 등을 활용해 해당 사이트의 파비콘/로고를 자동으로 불러와 표시
- **테마 및 디자인 커스텀:** 배경색, 버튼 스타일, 폰트 등을 변경할 수 있는 커스텀 옵션 제공

### 2.3. 추후 구현 과제 (Phase 2)
- **링크 클릭 통계 (Analytics):** 복잡한 추적이 아닌 **오직 각 링크별 '클릭 횟수(카운트)'**만을 추적하여 보여주는 직관적인 통계 기능 구현

## 3. 기능 상세 설명

### 3.1. 회원 인증 시스템
- **오직 Firebase 기반의 구글 소셜 로그인**만을 사용하여 간편하고 강력한 보안을 유지합니다. 아이디/비밀번호 가입은 지원하지 않습니다.
- 단일 사용자 전용 서비스이므로, 사전에 지정된 본인(개발자)의 구글 계정으로만 어드민(대시보드) 페이지에 접근 가능하도록 규칙을 설정합니다.

### 3.2. 링크 및 프로필 관리 (대시보드)
- **Firebase Firestore**를 데이터베이스로 사용하여, 입력된 링크와 프로필 정보를 실시간으로 저장하고 불러옵니다.
- 모바일과 데스크탑 환경 모두 동일하게, 텍스트를 클릭하여 바로 수정하는 **인라인 에디팅(Inline Editing)** 방식을 도입하되, 의도치 않은 저장을 방지하기 위해 반드시 **명시적인 '저장' 버튼**을 눌러야만 DB에 반영되도록 설계합니다.

### 3.3. 링크 노출 페이지 및 디자인 시스템
- **shadcn/ui** 컴포넌트를 적극 활용하여, 현대적이고 일관된 디자인 시스템을 신속하게 구축합니다.
- 모바일과 데스크탑 모두에서 완벽하게 보이는 **반응형 UI**로 구성하며, Firebase의 빠른 응답 속도를 활용해 최적의 뷰어 경험을 제공합니다.
